# 课程结构模式与校验

当需要选择结构类型、重构复杂大纲或诊断结构问题时，读取本参考。

## 结构类型选择

| 结构类型 | 适用课题 | 模块组织方式 | 常见风险 |
| --- | --- | --- | --- |
| 流程型 | 流程执行、项目管理、问题闭环、工具操作 | 按时间、步骤、阶段、操作顺序 | 只列步骤，不训练关键判断 |
| 要素型 | 能力模型、标准解读、检查清单、角色职责 | 按并列维度或评价要素 | 维度之间颗粒度不一致 |
| WWH | 理念推广、方法导入、管理议题、价值说明 | 为什么 -> 是什么 -> 怎么做 | 停留在宣导，缺少应用练习 |
| 案例问题型 | 质量问题、项目复盘、复杂决策、经验萃取 | 问题呈现 -> 分析 -> 方法 -> 应用 -> 复盘 | 案例热闹但没有方法沉淀 |
| 任务练习型 | 技能训练、模板填写、评审判断、沟通演练 | 任务1 -> 反馈 -> 任务2 -> 综合任务 | 练习之间缺少递进关系 |

## 常用结构模板

### 流程型

适合“如何完成一项工作”。

```text
1. 任务认知：目标、角色、输出和标准
2. 准备阶段：输入、资料、风险和条件
3. 执行阶段：关键步骤、判断点和工具
4. 输出阶段：成果物、检查标准和常见错误
5. 闭环阶段：反馈、改进和迁移应用
```

### 要素型

适合“一个标准/能力由哪些维度构成”。

```text
1. 要素一：定义、标准、典型表现
2. 要素二：定义、标准、典型表现
3. 要素三：定义、标准、典型表现
4. 综合应用：多要素判断或工作输出
```

### WWH

适合“先建立必要性，再理解方法，再应用”。

```text
1. Why：为什么需要这个方法/标准/能力
2. What：核心概念、原则、边界和判断标准
3. How：操作步骤、工具模板、场景应用
4. Transfer：真实任务练习和行动计划
```

### 案例问题型

适合“从问题中学方法”。

```text
1. 案例导入：事实、结果、冲突和风险
2. 问题识别：现象、偏差、影响和优先级
3. 原因分析：工具、证据、判断标准
4. 方案设计：措施、责任、资源和风险控制
5. 复盘迁移：方法总结、同类场景应用
```

### 任务练习型

适合“边做边学”。

```text
1. 标准建立：示范样例和评价标准
2. 单项任务：完成一个局部动作
3. 反馈修正：对照标准发现差距
4. 综合任务：完成完整工作输出
5. 迁移任务：换场景应用并总结原则
```

## 模块标题规则

好标题通常具备以下特征：

- 指向工作任务、判断、标准或输出。
- 能概括下一级内容。
- 与同级标题结构一致。
- 不依赖讲师解释也能看出学习方向。

弱标题示例：

| 弱标题 | 问题 | 改写方向 |
| --- | --- | --- |
| 背景介绍 | 太泛，不知道服务什么任务 | 复盘价值：从项目总结到改进闭环 |
| 模板讲解 | 资料名，不是学习任务 | 复盘输出：把问题转化为可跟踪行动项 |
| 案例分享 | 教学方法，不是内容结构 | 延期项目案例：识别事实、原因和行动 |
| 意识提升 | 口号，不可教学 | 复盘责任：避免追责式讨论，回到事实和改进 |

## 结构诊断清单

检查一份课程结构时，逐项判断：

- 是否能一句话说清主导结构逻辑。
- 一级模块是否覆盖全部课程目标。
- 同级模块是否处在同一抽象层级。
- 每个模块是否都有明确学习输出。
- 资料、案例、活动是否被误写成模块。
- 是否有重复模块或内容交叉。
- 是否存在没有目标支撑的背景内容。
- 是否有必须练习的技能却只安排讲授。
- 是否有态度目标却没有讨论、反思或决策场景。
- 是否有完整的学习递进：认知 -> 判断 -> 应用 -> 迁移。

## 重构步骤

当用户给出一份混乱大纲时，按这个顺序处理：

1. 提取所有内容项。
2. 删除明显不服务目标的内容。
3. 把资料名、活动名改写为内容任务。
4. 标记每项内容对应的目标或工作场景。
5. 合并重复项。
6. 选择主导结构逻辑。
7. 分组为一级模块和二级要点。
8. 排列顺序并写承接逻辑。
9. 标出可选内容和排除内容。
10. 输出结构校验结论。
