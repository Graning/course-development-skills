# 课程目标

## 概述

当用户需要编写课程目标、学习目标、培训目标、能力目标、模块目标，或希望把模糊目标改写成可衡量的学员成果时，使用这个能力。

课程目标要写成“学员完成课程后能够表现出的行为”，而不是讲师要讲什么、课程覆盖什么内容，或组织希望实现什么口号。一个好的目标会说明：学员能做什么、在什么条件下做、围绕什么内容做、达到什么标准。

## 核心公式

默认使用这个结构：

`对象 + 条件 + 动作 + 内容 + 标准`

其中：

- `对象`：学员群体、角色或参与者。
- `条件`：工作场景、给定材料、工具、案例或约束。
- `动作`：可观察的学员行为。
- `内容`：任务、知识对象、决策、输出物或工作成果。
- `标准`：质量要求、检查表、准确性、完整性、时限或验收标准。

当上下文已经明确时，可以使用短句形式：

`条件 + 学员能够 + 动作 + 内容 + 标准`

示例：

- 弱目标：`了解研发项目复盘方法。`
- 更好：`给定一个研发项目延期案例，学员能够区分事实、问题、原因和改进措施，并输出不少于3条可跟踪的复盘行动项。`

## 工作流程

### 1. 锚定工作绩效

写目标前，先识别：

- 目标学员和角色。
- 业务或绩效原因。
- 真实工作场景。
- 目标行为或输出物。
- 当前学员差距。
- 成功证据或评估方式。

如果这些信息缺失，可以保守推断并标记为假设。如果课题本身还不清楚，建议先使用 `$topic-analysis`。

### 2. 选择目标层级

先判断目标类别，再选择目标水平。默认使用三大类学习目标：

- `知识性目标`：学员需要知道、理解、解释、比较、判断或应用某类知识。
- `技能性目标`：学员需要演示、操作、制作、解决问题或熟练运用某项技能。
- `情感/态度性目标`：学员需要经历、感受、认同、支持、形成观点或表现出某种行为倾向。

再根据课程期望深度匹配目标水平：

- `知识性目标`：了解水平、理解水平、应用水平。
- `技能性目标`：模仿水平、独立操作水平、熟练操作水平。
- `情感/态度性目标`：经历/感受水平、反应/认同水平、领悟/内化水平。

如果课程只提供认知，不要写成高阶应用目标。如果学员最终必须完成工作任务，不要只使用弱动词。

选择动词和诊断弱目标时，读取 [objective-verbs.md](references/objective-verbs.md)。

### 3. 起草一组平衡目标

完整课程通常写 2-5 条目标：

- 1 条用于必要理解或关键原则。
- 1 条或多条用于核心任务执行。
- 如有需要，增加 1 条用于判断、场景处理或迁移应用。
- 只有当态度目标能通过讨论、决策、承诺或行为被观察时，才写态度目标。

保持目标颗粒度平行。不要在同一组目标里混合一个很小的知识点和一个完整岗位能力。

### 4. 让每条目标可检验

逐条检查：

- 讲师能否观察学员是否达成？
- 动作动词是否能产生可见证据？
- 内容是否具体到足以指导教学设计？
- 标准是否清楚到可以反馈？
- 是否匹配课程时长和教学方式？

如果答案是否定的，先修改目标，再进入下一步。

### 5. 诊断并改写弱目标

编辑已有目标时，先判断问题类型：

- `内容覆盖型`：写的是讲师要教什么，不是学员要做什么。
- `动词模糊`：了解、掌握、学习、熟悉、提升。
- `缺少条件`：没有场景、工具、案例或工作语境。
- `缺少标准`：没有质量要求或验收条件。
- `层级错误`：目标过宽、过窄，或超出课程可能达成的范围。
- `业务口号`：描述组织期待，不描述学员绩效。

输出时给出修改前/修改后，并简要说明改写原因。

## 输出格式

除非用户要求其他格式，按以下结构输出：

1. 输入假设：学员对象、场景、绩效目标和目标层级。
2. 课程目标组：2-5 条编号目标，使用可衡量表述。
3. 目标映射：目标 -> 目标类别/目标水平 -> K/A/S -> 证据/评估方式。
4. 修改说明：哪些内容被收窄、强化或保留为假设。
5. 待确认问题：只列会实质影响目标的关键输入。

快速请求时，只输出目标组和 1-2 条说明即可。

## 质量规则

- 优先使用 `学员能够...` 句式。
- 使用可观察动词；避免单独使用 `了解`、`掌握`、`熟悉`、`提升`、`增强`。
- 从学员视角写目标，不从讲师授课安排出发。
- 每条目标都要绑定工作场景、任务、输出、决策或标准。
- 每条目标都要能归入一种目标类别和一个目标水平。
- 行为可评估时，尽量写出标准。
- 目标要适配课程时长和教学方式。
- 必达目标放在可选目标或进阶目标之前。
- 目标不要过多；超过 5 条时，优先合并、分组或拆课。

## 与其他能力的关系

当目标学员、业务原因、工作场景或内容边界不清楚时，先使用 `$topic-analysis`。

目标稳定后，使用 `$course-structure` 搭建模块、层级和内容顺序。

结构稳定后，使用 `$content-organization` 组织课程内容；内容稳定后，使用 `$process-design` 继续设计教学流程、活动、案例、引导问题和验证方案。
