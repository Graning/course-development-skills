# 目标动词与写法模式

当需要选择动词、诊断弱目标或改写已有目标时，读取本参考。

## 三大类学习目标动词表

优先按“目标类别 -> 目标水平 -> 表现性动词”选择动词。

| 目标类别 | 目标水平 | 推荐使用的行为动词或参考格式 |
| --- | --- | --- |
| 知识性目标 | 了解水平 | 叙述、背诵、辨认、复述、描述、识别、列出、列举 |
| 知识性目标 | 理解水平 | 解释、阐明、比较、分类、区分、对……进行整理 |
| 知识性目标 | 应用水平 | 分析、设计、制订、选用、选择、判断、依据……解决……问题、检验 |
| 技能性目标 | 模仿水平 | 对……进行演示、模拟、编写、说出 |
| 技能性目标 | 独立操作水平 | 运用、使用、制作、操作、搭建、安装、开发、制定、解决、绘制 |
| 技能性目标 | 熟练操作水平 | 熟练操作、熟练运用、有效使用、依据……灵活运用、依据……进行转换 |
| 情感/态度性目标 | 经历/感受水平 | 对……进行体验、经历……过程、感受、讨论、观察、参与……尝试、意识到 |
| 情感/态度性目标 | 反应/认同水平 | 认可……并加以分享、识别、指出、支持……并说明理由、依据……做出……行为、分享 |
| 情感/态度性目标 | 领悟/内化水平 | 形成/确立/树立/构建……并做出……举动、具有……特点、表现……行为、坚持/保持……观点或行为 |

使用提醒：

- `了解水平` 不等于“了解”。目标表述中仍要写成可观察动作，如“列出、识别、描述”。
- `应用水平` 要写出依据、任务、问题或判断标准，否则容易退化成泛泛理解。
- `熟练操作水平` 要谨慎使用，只有课程时长、练习次数和评估方式足以支撑时才写。
- `情感/态度性目标` 不能只写“提升意识”，要写出讨论、选择、支持、承诺或表现出的行为。

## 补充动词库

| 类型 | 学员需要做到 | 推荐动词 |
| --- | --- | --- |
| 知识 | 回忆、识别或解释概念与规则 | 列出、识别、描述、解释、概述、区分 |
| 理解 | 比较观点、分类情境、解释逻辑 | 比较、分类、分析、梳理、解读、举例说明 |
| 技能 | 执行任务或产出工作成果 | 使用、完成、操作、演示、产出、起草、填写、组织 |
| 判断 | 按标准或在不确定情境中决策 | 诊断、评估、排序、选择、决定、建议、验证 |
| 问题解决 | 处理案例、问题、缺陷或项目风险 | 解决、排查、定位、纠正、追溯、制定 |
| 沟通 | 对齐、引导、提问或呈现 | 引导、提问、回应、呈现、协商、澄清、总结 |
| 态度 | 改变信念、责任意识或行为倾向 | 识别、讨论、反思、承诺、支持、选择、说明理由 |
| 迁移 | 应用到真实工作语境 | 应用、调整、改进、计划、实施、评审、优化 |

## 避免使用的弱动词

除非后面补充可观察证据，否则不要把这些词作为目标的主要动词：

- 了解
- 掌握
- 学习
- 知道
- 熟悉
- 提升
- 增强
- 强化
- 培养
- 建立意识

把它们改写成可见行为：

| 弱表达 | 更好的表达 |
| --- | --- |
| 了解审计要求 | 使用检查表识别审计案例中的缺失证据 |
| 掌握问题解决方法 | 使用 5 Why 输出符合评审标准的根因陈述 |
| 提升质量意识 | 说明不合格记录的风险影响，并选择必要纠正动作 |
| 学习项目评审流程 | 给定项目案例，完成阶段评审准备检查表 |

## 目标写法模式

### 知识目标

`完成学习后，学员能够说明/区分/解释 [概念或标准]，并结合 [场景] 判断 [正确/错误/风险]。`

示例：

`完成学习后，学员能够区分研发项目复盘中的事实、问题、原因和改进措施，并指出常见混淆点。`

### 技能目标

`给定 [案例/工具/模板]，学员能够 [动作] [工作输出]，达到 [标准]。`

示例：

`给定一个项目延期案例，学员能够使用复盘模板输出问题清单、原因分析和行动项，行动项包含责任人、完成时间和验证方式。`

### 判断目标

`面对 [复杂情境]，学员能够基于 [标准/原则] [判断动作]，并说明 [依据]。`

示例：

`面对多个项目问题，学员能够基于影响范围和复发风险确定优先复盘事项，并说明排序依据。`

### 态度目标

只有当态度能够被观察时，才写态度目标：

`通过 [案例/讨论/反思]，学员能够识别 [旧认知的问题]，并提出 [新的行为承诺或决策原则]。`

示例：

`通过复盘失效案例讨论，学员能够识别追责式复盘的风险，并提出3条保障事实讨论的会议规则。`

## 目标质量检查清单

逐条检查每个目标：

- 学员中心：描述学员行为，而不是讲师授课内容。
- 可观察：动词能够被看见、听见、读到或评价。
- 有场景：必要时写明案例、任务、工具、角色或工作语境。
- 具体：内容足够具体，能指导教学活动设计。
- 有标准：尽量包含质量标准或验收条件。
- 对齐：支撑课题边界和业务/绩效目标。
- 可达成：能在课程时长和教学方式内达成。

## 常见改写动作

- 增加工作条件：`给定...案例/模板/场景`。
- 替换模糊动词：`了解` -> `说明/区分/识别/判断`。
- 增加工作输出：`输出问题清单/完成检查表/形成改进项`。
- 增加标准：`符合...标准/包含...要素/能够说明依据`。
- 把过宽目标拆成两条：一条理解目标，一条应用目标。
- 把只列相邻知识点的小目标合并。
