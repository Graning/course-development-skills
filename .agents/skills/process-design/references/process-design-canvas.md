## 二、开场导入设计表

| 环节 | 时间 | AIDA 对应 | 开场五步法对应 | 教学目的 | 讲师动作/话术 | 学员动作 | 收口与衔接 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 问好 + 介绍 |  | Attention | 问好 + 介绍 | 建立课堂连接和讲师可信度，介绍只保留与课程主题相关的亮点经历 |  |  |  |
| 导入主题 |  | Attention / Interest | 导入主题 | 用提问、故事、案例、数据反差或情境判断引出核心问题 |  |  |  |
| 分组/破冰 |  | Interest | 分组/破冰 | 视人数、熟悉度和后续活动需要决定是否分组或破冰 |  |  |  |
| 公约 + 目标 |  | Desire / Action | 公约 + 目标 | 共同约定课堂注意事项，讲清课程目标和学习收益 |  |  |  |
| 课程结构 |  | Action | 课程结构 | 说明章节目录、重点内容和时间分配，形成学习地图并进入第一个模块 |  |  |  |

开场导入填写要求：

- 总时长不超过课程总时长的 5%-10%，用户另有限定时以用户要求为准。
- 每个问题、故事、案例、游戏或互动都必须绑定课程目标、学员痛点或课程大纲。
- 必须明确课程目标、核心收益、整体安排和课堂参与方式。
- 讲师介绍必须关联课程主题或内容，不做无关履历展示。
- 分组/破冰不是必做步骤；没有后续小组任务或学员已熟悉时可压缩或取消。
- 公约要与学习效果相关，例如手机、发言、讨论、时间、练习提交和反馈方式。
- 课程结构要说明章节目录、重点内容和时间分配。
- 结尾必须写清进入第一个模块的承接语。
- 不使用复杂道具；默认线下课堂可直接执行。

| 模块 | 对应目标 | 内容重点 | 学习难点 | 教学策略 | 讲师动作 | 学员动作 | 产出/衡量 | 时间 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  |  |  |  |

填写要求：

- `对应目标`：写课程目标编号，不写泛泛主题。
- `内容重点`：最多 1-3 个，避免把模块内容全量搬入。
- `学习难点`：写学员容易误解、不会做或不愿做的地方。
- `教学策略`：讲授、案例、讨论、示范、演练、工具练习、测验、复盘等。
- `产出/衡量`：必须可观察，例如答案、判断结果、表格、演示、行动项或测验结果。

活动安排要同时写清：

- 讲师发布什么任务。
- 学员个人、小组或全班分别做什么。
- 讲师如何收集结果。
- 讲师如何反馈和纠偏。
- 活动如何回扣课程目标。

## 五、教学策略选择提示

| 目标/内容类型 | 优先策略 | 必须保留的学习动作 |
| --- | --- | --- |
| 知识理解 | 问题导入、框架讲解、示例辨析、快速测验 | 说出、解释、辨认或判断 |
| 判断应用 | 情境案例、标准对照、小组分析、讲师点评 | 选择、分析、说明理由 |
| 技能操作 | 示范、模仿练习、反馈、二次练习 | 操作、演示、产出成果 |
| 态度转变 | 冲突体验、观点讨论、证据示证、行动承诺 | 表达观点、做出选择、承诺行动 |
| 迁移应用 | 真实任务、工具表、行动计划、同伴评审 | 形成可带回工作的输出 |

## 六、讲师引导清单

为以下节点准备可直接使用的问题或提示：

- 开场：这个课题和学员当前工作痛点有什么关系？
- 经验激活：学员过去遇到过什么典型问题？
- 内容输入：这个方法、标准或模型最容易被误解的点是什么？
- 案例分析：学员要观察什么、判断什么、依据什么？
- 练习反馈：好结果的标准是什么，常见偏差是什么？
- 模块承接：为什么从当前模块进入下一个模块？
- 收尾迁移：学员回到工作中第一步要做什么？

## 七、质量检查

- 目标是否都有对应活动和衡量证据？
- 核心内容是否都有应用或练习承接？
- 活动是否说明了讲师动作、学员动作和产出？
- 反馈是否被明确安排在练习之后？
- 时间是否优先给必达目标和高风险场景？
- 可选内容是否被压缩或下沉？
- 线上、线下或人数变化时是否有替代方案？
