# 过程设计

## 概述

当用户需要把课程结构和内容转化为完整授课流程、教案流程、教学活动、课堂互动、练习安排或讲师引导步骤时，使用这个能力。它位于课程设计顺序中的第 5 步：

`课题分析 -> 目标设定 -> 结构搭建 -> 内容组织 -> 过程设计`

过程设计不是把内容平均分配到时间表里，而是为每个课程目标选择合适的教学策略，让学员经历“进入状态、理解内容、尝试应用、获得反馈、形成迁移”的学习过程。

当用户要求“开场设计、主题导入、课程导入、破冰、开场讲稿、开场活动、导入方案”时，也使用本能力。此时重点不是热闹，而是用短时间完成课堂管理中的三件事：`期望值管理`、`注意力管理`、`学习动机管理`，让学员清楚“这门课与我有关、对我有用、接下来怎么学”。

本能力参考 `参考资料/aactp国际培训师认证-课程设计.pdf` 中的课程设计开发流程：在目标、结构和内容稳定后，设计教学过程、教学方法、活动安排和检验方式；同时沿用本项目课程设计体系中的 `内容 x 方法 = 效果`、KAS 分类和 PSAM 活动设计逻辑。

## 输入优先级

优先使用这些输入：

1. `$topic-analysis` 的学员对象、工作场景、绩效差距和 KAS 内容地图。
2. `$course-objectives` 的课程目标、目标水平和评估证据。
3. `$course-structure` 的模块顺序、模块承接逻辑和内容边界。
4. `$content-organization` 的模块内容、表达逻辑、案例素材、练习任务和工作应用提示。
5. 用户给出的课程时长、授课形式、人数、场地、工具、讲师限制或交付模板。

如果用户专门要求课程开场导入，必须优先收集或从已有材料中提取这些输入；没有核心输入时不直接生成完整导入方案：

1. `课程完整主题`：课程全称，不能只写宽泛领域。
2. `课程总时长`：用于控制导入时长红线。
3. `最终版课程核心目标`：课程整体目标或目标清单。
4. `课题分析核心结论`：目标学员画像、工作场景、核心痛点、能力短板或“不懂/不会/不愿”。
5. `课程大纲核心框架`：至少包含一级模块，最好能看到第一个模块。
6. `导入基础要求`：导入限定时长、授课人数、行业/岗位、线上线下、场地或道具限制。

如果缺少课程目标、模块结构或内容组织，先做必要假设并标注；如果缺口会明显影响活动和时间安排，建议先补 `$course-objectives`、`$course-structure` 或 `$content-organization`。

如果是开场导入请求，且缺少课程主题、课程目标、学员痛点或课程大纲核心框架，不生成完整方案；先向用户索要缺失材料，或只输出“需要补充的输入清单”。

## 核心流程

### 1. 确定过程设计约束

先确认会影响课堂流程的硬约束：

- `课程时长`：总时长、每个模块可用时间、休息节奏。
- `授课形式`：线下、线上、混合、讲授、工作坊、案例研讨或实操训练。
- `学员规模`：人数决定互动密度、分组方式和反馈方式。
- `学习产出`：课堂结束时必须形成的判断、练习结果、工具表或行动计划。
- `资源限制`：案例、数据、工具、模板、设备、场地和助教支持。

如果约束不明，默认设计为企业内训常见的 2-4 小时线下或线上同步课程，并标注假设。

### 2. 匹配教学策略

根据内容类型和目标水平选择教学策略：

- `知识理解`：引发问题 -> 讲解框架 -> 示例辨析 -> 快速确认。
- `判断应用`：情境案例 -> 小组分析 -> 标准对照 -> 讲师点评。
- `技能操作`：示范步骤 -> 学员尝试 -> 反馈修正 -> 二次练习。
- `态度转变`：冲突体验 -> 观点表达 -> 证据示证 -> 行动承诺。
- `迁移应用`：真实任务 -> 工具产出 -> 同伴评审 -> 回岗计划。

不要用单一讲授覆盖所有内容。知识点可以讲清，判断和技能必须让学员做出来，态度目标必须让学员表达、选择或承诺。

### 3. 设计主题导入/开场

当任务包含开场、导入或破冰时，单独设计开场环节。开场不是独立表演，必须从学员真实工作痛点出发，精准引出课程主题、课程目标和第一个模块。

#### 开场主线绑定

导入设计必须 100% 绑定以下内容：

- `课程目标`：开场要让学员知道本课最终要解决什么问题、形成什么能力或产出。
- `学员痛点`：导入素材、问题、故事、案例或互动必须命中学员工作中的高频痛点。
- `课程大纲`：开场必须自然引出课程整体安排，并平滑过渡到第一个核心模块。
- `课堂管理`：开场要同时完成期望值管理、注意力管理、学习动机管理。

不得设计与课程主线无关的游戏、故事、热身、破冰或鸡汤式发言。不能解释清楚“为什么这个导入能服务目标和痛点”的设计，应删除或重写。

#### AIDA 开场原则

开场默认按 AIDA 组织：

- `Attention 吸引注意力`：用痛点问题、真实场景、反差数据、错误案例、快速判断题或情境投票抓住注意力。
- `Interest 引发兴趣`：让学员看见自己工作中的困扰、风险、成本或机会。
- `Desire 传递价值`：说明学完后能解决什么问题、做出什么成果、减少什么代价。
- `Action 明确安排`：说明课程目标、学习路径、参与方式、时间安排和第一个模块入口。

#### 开场五步法

除非用户另有模板，开场按五步法设计：

1. `建立连接`：用与学员岗位相关的情境、问题或观察开场。
2. `痛点共鸣`：让学员确认自己确实遇到过这个问题，而不是讲师单方面定义需求。
3. `价值承诺`：把课程目标转译成学员能感知的工作收益。
4. `规则与安排`：说明课程结构、参与方式、互动要求、时间边界和产出要求。
5. `导入正文`：用一句承接语进入第一个核心模块。

#### 开场时长红线

严格控制导入总时长：

- 导入总时长一般不超过课程总时长的 `5%-10%`。
- 1 小时课程建议开场 `3-6 分钟`。
- 半天课程建议开场 `10-20 分钟`。
- 1 天课程建议开场 `20-40 分钟`。
- 如果用户给出导入限定时长，以用户限制为准，并拆分到每个环节。

如果设计超过时长红线，必须主动压缩互动复杂度、讲师铺垫和背景说明。

#### 开场形式选择

根据学员行业、岗位、人数和场地限制选择导入形式：

- `痛点提问`：适合短课、大班、无道具场景。
- `情境投票`：适合需要快速唤起共识的课程。
- `真实案例/失败案例`：适合流程、质量、管理、销售、项目类课程。
- `快速诊断`：适合工具方法、能力提升、问题解决类课程。
- `任务挑战`：适合技能训练、实操工作坊。
- `微型讨论`：适合 15-40 人线下班，需控制时间和汇报人数。

优先选择无需复杂道具、线下课堂可直接执行的形式。互动不能只写“请大家讨论”，必须写清讲师怎么问、学员怎么做、如何收口、如何导向课程主题。

### 4. 设计模块流程

每个模块默认按以下节奏设计：

`承接目标 -> 激活经验 -> 输入关键内容 -> 练习/讨论/案例 -> 反馈澄清 -> 小结迁移`

为每个模块明确：

- 这个模块服务哪条课程目标。
- 学员进入模块前需要被唤起的经验或问题。
- 讲师要讲清的 1-3 个关键点。
- 学员必须完成的活动、判断、演练或产出。
- 讲师如何观察学习是否发生。
- 本模块如何承接前后模块。

模块流程要顺着学习难度推进：先建立共识和标准，再进入判断、操作、综合应用。

### 5. 用 PSAM 设计活动

每个活动都用 PSAM 校准：

- `Purpose（目的）`：活动服务哪条目标、哪个内容难点或哪个学习状态。
- `Strategy（策略）`：讲授、讨论、案例、演练、示范、复盘、测验、工具练习等。
- `Activities（活动）`：讲师做什么，学员做什么，个人/小组/全班如何组织。
- `Measure（衡量）`：用什么问题、答案、作品、演示、检查表或反馈确认学习结果。

活动不是气氛装饰。不能说明教学目的和衡量方式的活动，应删除或改写。

### 6. 安排时间和节奏

按学习价值分配时间，而不是按内容字数平均分配：

- 必达目标和高风险工作场景优先给时间。
- 技能、判断和迁移应用要预留练习与反馈时间。
- 背景介绍、制度说明和低风险知识点要压缩。
- 每 15-25 分钟应有一次学习动作变化：提问、判断、演练、讨论、输出或反馈。
- 开场必须完成 AIDA 和期望值管理，收尾必须服务巩固、迁移和行动承诺。

如果总时长不足，优先压缩背景和可选知识，不压缩核心练习反馈。

### 7. 设计讲师引导

为关键节点写出讲师引导，而不是只写“互动讨论”：

- `开场问题`：让学员意识到课程和工作场景相关。
- `承接语`：说明为什么进入下一个模块。
- `引导问题`：推动学员观察、比较、判断、总结。
- `反馈话术`：指出标准、偏差和改进方向。
- `转化问题`：让学员把课堂内容迁移到自己的工作。
- `收尾动作`：让学员输出行动计划、检查清单或承诺。

引导语言应简洁可执行，避免写成泛泛主持词。

开场讲稿必须贴合线下课堂授课语气，口语化、场景化，可直接照着讲。必要时标注 `[停顿]`、`[举手/投票]`、`[板书]`、`[语气加重]`、`[转身看屏幕]`、`[收口]` 等提示，但不要写成舞台剧脚本。

### 8. 校验过程有效性

最终检查：

- 每条课程目标都有教学活动和衡量证据承接。
- 每个活动都服务内容和目标，而不是单纯活跃气氛。
- 讲师动作、学员动作、材料和产出清楚。
- 模块顺序符合学习难度和工作逻辑。
- 知识输入、练习、反馈和迁移时间比例合理。
- 必讲内容和必练内容没有被可选内容挤压。
- 线上或线下形式下都能真实执行。
- 开场是否绑定课程目标、学员痛点和课程大纲，而不是独立热身。
- 开场是否覆盖 AIDA 和开场五步法。
- 开场是否完成期望值管理、注意力管理和学习动机管理。
- 导入结尾是否自然衔接第一个模块。

详细表格模板见 [process-design-canvas.md](references/process-design-canvas.md)。

## 输出格式

除非用户要求其他格式，按以下结构输出：

1. 过程设计假设：时长、形式、人数、材料和限制。
2. 总体流程：开场、模块顺序、收尾和时间分配。
3. 模块过程表：模块、目标、内容重点、教学策略、讲师动作、学员动作、产出/衡量、时间。
4. 活动设计表：活动名称、PSAM、组织方式、材料、反馈方式。
5. 讲师引导：开场问题、模块承接语、关键提问、点评要点、收尾动作。
6. 风险与调整：时间不足、材料缺口、线上/线下调整、人数变化时的替代方案。
7. 过程校验：目标覆盖、练习反馈、学习证据和迁移应用。

如果用户专门要求“开场导入/主题导入”，按以下结构输出：

1. `导入核心目标`：说明导入要完成的注意力、动机、期望值和课程主线任务。
2. `导入形式与总时长`：说明选用形式、适配原因、总时长，并确认不超过课程总时长 5%-10%。
3. `完整讲稿`：包含讲师话术、讲师动作、互动设计、学员回应方式、收口话术和进入第一模块的衔接语。
4. `分环节时间分配`：用表格拆分每一步，包含时间、AIDA 对应、五步法对应、讲师动作、学员动作、产出。
5. `设计亮点与课堂管理价值`：说明如何完成期望值管理、注意力管理、学习动机管理，以及如何贴合课程目标和学员痛点。

快速请求时，可只输出“总体流程 + 模块过程表 + 关键引导问题”。

## 质量规则

- 先看目标和内容类型，再选教学方法。
- 过程设计必须服务学员学习，不服务讲师讲稿堆叠。
- 不把“讲授、讨论、互动”写成空泛活动；必须说明学员做什么和产出什么。
- 知识内容要有确认，技能内容要有练习，态度内容要有表达或承诺。
- 核心练习必须包含反馈，不只安排任务。
- 每个模块都要有承接和小结。
- 时间分配要体现重点，不能平均主义。
- 如果课程时长不足，主动提出删减或下沉可选内容。
- 开场导入必须从学员真实工作痛点进入，不能做与课程目标无关的破冰。
- 开场必须明确告知课程目标、核心收益和整体安排。
- 开场互动要能被讲师收口，不能把课堂带到与主线无关的方向。
- 开场讲稿要口语化、可执行，避免书面化、空泛化、模板化。

## 与其他能力的关系

使用 `$topic-analysis` 先明确课题、对象、场景和 KAS 差距。

使用 `$course-objectives` 先明确可观察课程目标和评估证据。

使用 `$course-structure` 先稳定模块、层级和顺序。

使用 `$content-organization` 先完成内容筛选、表达逻辑和应用连接。

使用 `$process-design` 把稳定内容转化为课堂流程、教学活动、讲师引导和学习衡量。
