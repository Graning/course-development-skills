# 课题分析

## 概述

在编写课程大纲、教案、PPT、案例或培训材料之前，使用这个能力。它的目标是把一个模糊课题转化为一个站得住的课程简报：为什么要做、谁需要改变行为、服务哪个工作场景、哪些内容应该纳入范围、哪些内容应该排除。

输出应保持分析性质。除非用户明确要求，不要直接进入完整课程设计；只有在课题边界和学习问题清楚之后，再进入 `$course-objectives`。

## 核心流程

### 1. 澄清业务原因

从业务结果或质量管理结果出发，而不是从讲师已有材料清单出发。需要捕捉：

- `业务目标`：期望改善什么运营结果、质量结果、合规结果或项目结果。
- `触发原因`：为什么现在需要这个课题。
- `成功信号`：什么可见变化能说明培训产生了效果。
- `非培训因素`：流程、制度、工具、人员配置、激励或管理问题中，哪些不是培训本身能单独解决的。

如果业务原因缺失，可以推断一个可能原因，但必须标记为假设，不要把推断写成确定事实。

### 2. 定义学员和场景

按角色、经验水平、任务责任和当前压力识别学员群体。然后把课题锚定到 1-3 个具体工作场景：

`角色 -> 工作任务 -> 决策/动作 -> 输出物 -> 标准或风险`

优先使用真实工作任务，而不是泛泛的能力词。对于质量体系课程，可以关注审计准备、问题关闭、流程执行、文件评审、偏差处理、变更控制、供应商管理、项目里程碑评审等场景。

### 3. 诊断绩效差距

区分表面症状和背后原因：

- `可见问题`：学员现在做错、漏做、延迟、误解或抗拒的行为。
- `目标行为`：好的绩效表现应该是什么样，必须可观察。
- `差距类型`：知识差距、技能差距、态度差距、工具/流程差距，或管理/环境差距。
- `不改变的风险`：质量、成本、进度、合规、客户、安全或协同影响。

使用这条分析链：

`工作任务场景 -> 关键行为/动作 -> 可见问题 -> 隐性内容 -> 课程内容`

### 4. 用 KAS 提取隐性内容

对弥补差距所需的隐性内容做分类：

- `K`：概念、标准、规则、判定准则、案例、决策原则。
- `A`：信念、风险意识、责任意识、动机、利益相关方视角。
- `S`：流程操作、判断、沟通、执行、评审、问题解决。

不要把所有课题都当成知识课。如果最终行为需要判断或执行，分析中必须包含练习场景和反馈方式的启示。

### 5. 设定范围和边界

明确课程应该覆盖和不应该覆盖的内容：

- `必须覆盖`：直接支撑目标任务和目标行为的内容。
- `可选内容`：有帮助但可以压缩、作为预习材料，或放到附录中的内容。
- `排除内容`：背景知识、边缘情况、工具细节、制度内容，或与学员目标绩效无直接关系的管理问题。

优先选择更窄、更可教学的课题，而不是宽泛主题。如果一个课题包含多个互不相关的工作任务，建议拆分成不同课程或模块。

### 6. 转化为课程启示

最后输出课题分析简报，而不是完整课程设计：

- 课程定位和建议标题方向。
- 目标学员和前提假设。
- 2-4 条可能的课程目标，使用可衡量行为表达。
- 推荐结构逻辑：`流程型`、`要素型`、`WWH`、`案例问题型` 或 `任务练习型`。
- 内容优先级和排除项。
- 后续需要的证据、案例、练习、工具表或来源材料。

当用户请求比较模糊、课题风险较高，或产出将作为正式课程开发材料时，读取 [analysis-canvas.md](references/analysis-canvas.md)，使用其中的详细画布和校验清单。

## 输出格式

除非用户要求其他格式，按以下结构输出：

1. 课题判断：这是适合做成课程的课题、模块课题、材料课题，还是非培训问题。
2. 业务与绩效背景：目标、触发原因、成功信号、非培训约束。
3. 学员与场景分析：对象、工作场景、任务输出、标准/风险。
4. 差距诊断：可见问题、目标行为、差距类型、根因假设。
5. KAS 内容地图：K/A/S 条目、优先级和所需证据。
6. 范围边界：必须覆盖、可选内容、排除内容。
7. 课程启示：标题方向、课程目标、结构逻辑、活动/案例启示。
8. 待确认问题：只列出会实质影响分析结论的关键缺失输入。

## 质量规则

- 每一个建议内容都必须绑定到学员任务或绩效差距。
- 当来源材料不足时，明确标记假设。
- 不要保留宽泛口号作为模块标题；要转化成任务、决策、标准或输出物语言。
- 明确指出哪些问题不是培训单独能解决的。
- 必须内容排在可选内容之前。
- 如果用户提供了现有文档，应从文档中提取真实学员任务和字段标准，而不是泛泛总结。

## 与其他能力的关系

使用 `$topic-analysis` 来确定课程原因、边界、对象、场景、差距和内容地图。

课题边界清楚后，使用 `$course-objectives` 设定可衡量课程目标；目标稳定后，再依次进入 `$course-structure`、`$content-organization` 和 `$process-design`。
