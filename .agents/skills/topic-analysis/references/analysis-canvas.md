# 课题分析画布

当课题模糊、范围很宽、牵涉协同或合规风险，或产出将作为正式课程开发材料时，使用本参考。

## 快速访谈表

| 字段 | 提问 | 说明 |
| --- | --- | --- |
| 课题 | 拟定课程/课题名称是什么？ | 先保留原始说法，再优化成工作语言。 |
| 发起方目标 | 发起方期望改善什么业务或质量结果？ | 优先使用可衡量结果或判断标准。 |
| 触发原因 | 为什么现在要做？ | 新流程、重复问题、审计发现、项目失败、角色变化等。 |
| 学员对象 | 谁必须表现得不一样？ | 角色、资历、经验、地点、职能。 |
| 工作场景 | 学员会在什么工作情境中使用？ | 描述真实任务和输出物。 |
| 当前问题 | 学员现在做错或没有做到什么？ | 只写可观察行为。 |
| 标准 | 好的表现是什么样？ | 检查表、模板、SOP、评审标准、案例标准。 |
| 证据 | 什么证明差距存在？ | 数据、审计发现、主管反馈、缺陷、延期、返工。 |
| 约束 | 哪些问题不是培训能解决的？ | 工具、流程、权限、激励、工作量、职责不清。 |

## 课题类型判断

设计前先判断课题类型：

- `课程课题`：支撑一个反复发生的角色任务，需要多个目标、练习和迁移。
- `模块课题`：适合放在更大课程中，范围窄但可教学。
- `材料课题`：主要是参考资料、工具表、SOP 说明、FAQ 或检查表。
- `沟通课题`：更需要通知、对齐或变更沟通，而不是培训。
- `管理/系统问题`：需要流程、权限、资源或治理先调整，培训才可能有效。

如果课题不适合做成课程，说明更合适的形式，并简要解释原因。

## 场景到内容的分析链

每个核心场景至少建立一条链：

| 工作任务场景 | 关键动作 | 可见问题 | 隐性内容 | KAS | 优先级 | 课程启示 |
| --- | --- | --- | --- | --- | --- | --- |
|  |  |  |  | K/A/S | 必须/可选 | 目标、模块、案例、练习或工具表 |

规则：

- `工作任务场景` 必须是真实工作情境，不是概念。
- `关键动作` 必须可观察。
- `可见问题` 必须描述当前行为。
- `隐性内容` 必须解释学员缺什么或误解什么。
- `课程启示` 必须说明课程应如何处理。

## KAS 诊断指南

当学员需要以下内容时，归为 `K`：

- 术语、原则、标准、制度、检查表。
- 正确与错误样例对比。
- 决策标准或风险逻辑。

当学员需要以下内容时，归为 `A`：

- 责任意识、质量意识、客户/利益相关方视角。
- 风险意识，或按标准执行的意愿。
- 挑战、升级或沟通的信心。

当学员需要以下内容时，归为 `S`：

- 反复练习、判断、评审、操作、沟通、引导或问题解决。
- 基于标准的反馈。
- 把方法迁移到自己的工作输出。

如果目标结果是 `S`，不要建议只做讲授型课程。

## 范围筛选

用这些问题裁剪内容：

- 这个内容是否直接支撑目标工作场景？
- 学员是否对这个内容负有行动责任？
- 这是首次正确完成任务所必需的，还是专家级边缘情况？
- 能否移到预习、附录、工具表或主管说明中？
- 删除它是否会削弱目标行为？

给每项内容打标签：

- `必须`：正确完成目标任务所必需。
- `可选`：有帮助，但课堂内不是必要。
- `排除`：不需要、对象不对，或属于非培训问题。

## 目标草拟

课题分析阶段只草拟初步目标。使用：

`条件 + 学员动作 + 工作内容 + 表现标准`

示例：

- `给定一个质量问题场景，学员能够按 CAPA 评审标准区分现象、原因、纠正措施和预防措施。`
- `使用项目评审检查表，学员能够识别缺失的阶段评审证据，并说明需要补充的内容。`
- `面对供应商偏差案例，学员能够选择升级路径，并解释风险依据。`

## 校验清单

最终确认前，检查：

- 业务目标已经明确，或已标记为假设。
- 学员对象确实承担目标工作任务。
- 至少命名了一个真实工作场景。
- 目标行为可观察。
- K/A/S 分类与绩效差距匹配。
- 范围排除了只属于背景或对象不匹配的内容。
- 非培训约束已经说明。
- 课程启示足够具体，能继续进入 `$course-objectives`。
