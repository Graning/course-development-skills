# 试题生成

## 概述

当用户需要根据课程材料、课件内容、课程目标、课程大纲或教学重难点生成课后考试题、随堂测试题、培训笔试试卷、答案解析或题库时，使用这个能力。

本能力服务企业培训课程开发的学习评估环节，重点检验学员是否掌握课程目标要求的知识、技能和态度认知。它不是为了凑题，而是通过题目设计帮助讲师确认学习效果、强化核心知识点、暴露常见误区。

默认采用“紧扣课程目标，分层分级出题”的高质量模式；如果用户明确要求“简单测试、记忆型题、随堂测”，可使用快速记忆型模式。

## 输入要求

优先使用以下核心输入。正式课后考试题缺少关键输入时，不直接生成完整试卷，应先提示用户补充：

1. `课程完整主题`：课程全称。
2. `最终版课程目标`：最好区分知识、技能、态度三类目标；若来自 `$course-objectives`，优先使用目标编号。
3. `课程核心教学内容与重难点`：核心知识点、技能点、教学重难点、学员高频易错点。
4. `试卷基础要求`：总题量、题型占比、总分值、每题分值、考试时长。
5. 可选输入：学员画像、岗位场景、课件正文、课程结构、内容组织结果、题目难度偏好、是否用于线上考试或纸笔考试。

如果用户只提供课件或课程材料，但没有试卷要求，默认建议：

- 20 道题。
- 单选题 15 道，多选题 5 道。
- 每题 5 分，总分 100 分。
- 适合随堂测试或课后基础验收。

如果用户提供了企业内部导入模板或题库格式，严格按模板字段输出。

## 核心规则

### 1. 主线绑定

所有题目必须来自课程目标、核心教学内容和重难点。不得出现课程里未覆盖的超纲题、偏题、怪题，也不得为了凑数量生成与岗位工作无关的理论题。

题目权重必须贴合教学重难点：

- 核心知识点、关键技能点、学员高频易错点题量占比不低于 70%。
- 非核心背景知识只能少量出现，不能挤占核心目标题量。
- 每道题都要能追溯到课程目标、模块或具体知识点。

### 2. ASK 匹配

按课程目标类型选择题目考法：

- `知识类目标`：优先用单选题、多选题、判断题，考察记忆、理解、辨认和概念边界。
- `技能类目标`：优先用案例分析题、情景题、流程排序题、场景选择题，考察应用、判断和操作步骤。
- `态度类目标`：优先用场景判断题、价值取舍题、行为选择题，考察认知认同和行为倾向。

不要把技能目标写成纯概念记忆题。不要把态度目标写成口号题。

### 3. 难度梯度

默认难度结构：

- `基础题 60%`：检验核心概念、流程、标准和关键要求。
- `进阶题 30%`：检验典型场景下的判断与应用。
- `拔高题 10%`：检验复杂场景、易错点辨析或综合应用。

如果用户指定难度比例，以用户要求为准；但必须保证基础题能覆盖必达目标，进阶和拔高题能区分掌握程度。

### 4. 题型质量

题干要简洁、清晰、贴合岗位真实工作场景。选项设计要逻辑严谨：

- 单选题只能有一个最佳答案。
- 多选题必须明确“多选、少选、错选均不得分”。
- 错误选项应贴合学员常见误区，而不是明显荒谬。
- 选项之间应同类、同层级、长度尽量接近。
- 避免“以上都对/以上都不对”，除非课程评估确实需要。
- 避免模棱两可、双重否定、暗示答案的题干。

### 5. 答案与解析

每道题都必须给出答案和解析。解析不仅说明正确答案为什么对，也要说明关键错误选项为什么错，并标注对应的课程目标、课程模块或知识点。

解析要服务“以考促学、以考固学”，不能只写“根据课件可知”。

## 生成流程

1. `解析输入`：提取课程主题、目标、模块、核心内容、重难点、易错点和题量分值要求。
2. `建立命题蓝图`：按目标、模块、ASK 类型、难度和题型分配题量。
3. `生成试题`：先覆盖必达目标和高频易错点，再补充进阶和拔高题。
4. `生成答案解析`：逐题标注答案、解析、对应目标/知识点、难度。
5. `核算合规`：检查题量、题型、分值、总分、难度比例、目标覆盖率和是否超纲。
6. `输出试卷`：按用户指定格式输出；未指定时用标准文字试卷格式。

正式试卷建议先输出“命题蓝图”，再输出试题。快速请求可直接输出完整试卷。

## 输出格式

除非用户要求其他格式，正式试卷按三大模块输出：

1. `试卷说明`
   - 课程主题
   - 考试对象
   - 考试时长
   - 题型与分值
   - 总分
   - 作答规则，尤其是多选题得分规则

2. `试题部分`
   - 按题型分组，例如单选题、多选题、判断题、案例题。
   - 每题包含题干、选项、分值。
   - 题目编号连续，题型内编号清晰。

3. `答案与解析部分`
   - 题号
   - 正确答案
   - 解析
   - 对应课程目标/模块/知识点
   - 难度：基础/进阶/拔高

如用户需要 Excel、企业微信在线文档或题库导入格式，先确认字段要求，再按字段输出或生成文件。

## 快速记忆型模式

当用户明确要求“仅针对课件内容生成简单记忆型考试题、随堂测试、快速测验”时：

- 以课件原文和核心知识点为主，不强行加入复杂案例。
- 默认 20 题：15 道单选题、5 道多选题，每题 5 分，总分 100 分。
- 仍需提供答案和简要解析。
- 不生成课件未覆盖内容。

## 质量检查

输出前逐项检查：

- 题目是否 100% 来自课程目标、课件内容和教学重难点。
- 核心知识点和高频易错点题量是否达到 70% 以上。
- 知识、技能、态度目标是否有对应题目。
- 难度比例是否接近 60% / 30% / 10%。
- 题型、题量、分值和总分是否核算准确。
- 单选题是否只有一个最佳答案。
- 多选题答案是否无歧义，并注明得分规则。
- 错误选项是否贴合常见误区。
- 每道题是否有答案、解析和目标/知识点标注。
- 是否适配目标学员的行业、岗位和真实工作场景。
