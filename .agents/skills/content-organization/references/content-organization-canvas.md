# 内容组织画布

当用户提供大量资料、专家经验、制度文本、PPT 或零散素材时，使用本画布。

## 内容取舍表

| 来源内容项 | 对应目标 | 对应模块 | 处理方式 | 理由 |
| --- | --- | --- | --- | --- |
|  |  |  | 必讲/必练/可选/排除 | 说明与目标、场景或应用的关系 |

处理方式定义：

- `必讲`：支撑理解、判断或应用的必要内容。
- `必练`：对应技能、判断或迁移应用，必须通过练习或成果物确认。
- `可选`：有帮助但不影响目标达成，可压缩、预习或放附录。
- `排除`：与目标学员、工作场景或课程目标无关。

## 内容分类表

| 分类 | 内容说明 | 适合的教学处理 |
| --- | --- | --- |
| 通用原理 | 概念、模型、标准、规则、判断原则 | 讲授、提问、图示、对比样例 |
| 实践经验 | 专家做法、成功/失败经验、常见误区 | 案例、故事、经验萃取、对照分析 |
| 操作方法 | 步骤、流程、模板、工具、检查表 | 示范、演练、工具练习、纠错 |
| 案例素材 | 真实事件、典型问题、冲突和结果 | 决策案例、分析案例、小组讨论 |
| 练习任务 | 判断、产出、演练或应用任务 | 个人练习、小组任务、角色演练 |
| 工作启发 | 应用场景、风险提醒、改进建议 | 行动计划、迁移讨论、工作清单 |

## 知识结构梳理

把每个模块中的内容按这条链整理：

`上位概念/模块主题 -> 关键知识点 -> 判断标准/步骤方法 -> 正反例 -> 工具/模板 -> 工作应用`

检查：

- 上位主题能否概括下位内容。
- 同级知识点颗粒度是否一致。
- 概念是否有边界。
- 方法是否有步骤。
- 标准是否可判断。
- 案例是否贴近工作。
- 应用是否指向真实任务。

## 表达逻辑模板

### 模板一：是什么 -> 为什么 -> 怎么做

适合方法、原则、制度要求、关键标准。

| 环节 | 要回答的问题 |
| --- | --- |
| 是什么 | 这个概念、方法或要求是什么？边界是什么？ |
| 为什么 | 为什么它重要？不这样做有什么风险或损失？ |
| 怎么做 | 学员在工作中应该按什么步骤、标准或工具执行？ |

### 模板二：是什么 -> 有什么 -> 怎么做

适合模型、要素、能力维度、检查清单。

| 环节 | 要回答的问题 |
| --- | --- |
| 是什么 | 这个模型、标准或工具是什么？ |
| 有什么 | 它包含哪些要素、维度、指标或常见类型？ |
| 怎么做 | 学员如何使用这些要素完成判断或输出？ |

## 内容组织检查清单

- 内容是否与主题一致。
- 内容是否连接目标学员。
- 内容之间是否有关联，而不是散点堆放。
- 是否补充了对工作的启发和应用。
- 是否区分了课堂主线内容和附录材料。
- 是否把专家经验提炼成可迁移方法。
- 是否为技能/判断内容设计了练习或案例。
- 是否能支持后续教学过程设计。
