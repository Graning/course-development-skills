# 内容组织

## 概述

当用户需要从资料、经验、制度、案例或零散素材中整理课程内容时，使用这个能力。它位于课程开发顺序中的第 4 步：

`课题分析 -> 目标设定 -> 结构搭建 -> 内容组织 -> 过程设计`

内容组织不是简单汇总资料，而是依据课程目标进行裁剪，把“要讲的东西”整理成“学员为了完成工作任务必须理解、判断、练习和应用的内容”。

本能力参考 `参考资料/aactp国际培训师认证-课程设计.pdf` 中的课程设计开发流程：课程目标为内容裁剪提供依据；组织培训内容时要处理通用原理、实践经验、知识结构梳理和表达逻辑建立。

## 输入优先级

优先使用这些输入：

1. `$topic-analysis` 的课题边界、学员场景、问题和 KAS 内容地图。
2. `$course-objectives` 的课程目标和目标水平。
3. `$course-structure` 的模块结构、模块承接逻辑和内容边界。
4. 用户给出的 SOP、制度、模板、PPT、案例、专家访谈、经验材料或素材清单。

如果课程目标或模块结构缺失，先做必要假设并标注；如果缺口会影响内容取舍，建议先使用 `$course-objectives` 或 `$course-structure`。

## 核心流程

### 1. 用课程目标裁剪内容

先把所有素材拆成内容项，再逐项判断是否服务课程目标：

- 是否直接支撑某条课程目标？
- 是否服务目标学员的真实工作任务？
- 是否能帮助学员解决可见问题或完成目标行为？
- 是否需要在课堂中讲授，还是更适合预习、附录、工具表或课后阅读？

给每项内容标注：

- `必讲`：没有它，学员无法达成目标。
- `必练`：它对应技能、判断或迁移应用，必须安排练习或产出。
- `可选`：有帮助但不影响目标达成，可压缩或下沉。
- `排除`：与对象、目标或场景无关，或属于非培训问题。

### 2. 区分内容类型

把保留下来的内容按类型归类：

- `通用原理`：概念、模型、标准、规则、判断原则。
- `实践经验`：专家做法、成功/失败经验、常见误区、关键提醒。
- `操作方法`：步骤、流程、模板、工具、检查表。
- `案例素材`：真实事件、典型问题、情境、冲突、结果。
- `练习任务`：学员需要完成的判断、产出、演练或应用任务。
- `工作启发`：对真实工作的应用、改进、风险提醒或迁移建议。

内容不能只停留在“是什么”，还要说明“为什么重要、怎么做、对工作有什么启发”。

### 3. 梳理知识结构

把零散内容组织成可讲、可学、可记的知识结构：

- 合并重复和相近内容。
- 把上位概念、下位知识点、案例和工具分层。
- 明确每个内容项在模块中的位置。
- 保持同一层级的内容颗粒度一致。
- 为关键知识点补充定义、判断标准、常见错误和工作示例。

当内容来自资料目录时，不要照搬资料标题；要改写成学习任务、判断标准、操作方法或工作输出。

### 4. 建立表达逻辑

为每个关键知识点建立表达逻辑。优先使用这两种：

- `是什么 -> 为什么 -> 怎么做`：适合方法、原则、标准、制度要求。
- `是什么 -> 有什么 -> 怎么做`：适合模型、要素、能力维度、检查清单。

表达时至少包含：

- `关键词`：这个内容的核心概念或要点。
- `解释`：它是什么意思，边界是什么。
- `观点`：讲师希望学员形成的判断。
- `举例`：贴近学员工作的正反例或典型场景。
- `应用`：学员回到工作中如何使用。

### 5. 连接工作应用

检查每个内容项是否能落到工作应用：

- 学员用它解决什么问题？
- 学员用它完成什么输出？
- 学员在哪个场景中会用到？
- 学员需要判断什么、避免什么错误？
- 是否需要设计案例、练习或工具表来承接？

缺少“对工作的启发”和“应用”的内容，要么补齐应用说明，要么下沉为背景资料。

### 6. 输出内容组织表

最终输出应让后续 `$process-design` 能直接进入过程设计。每个模块下都要清楚列出：讲什么、为什么讲、怎么表达、如何应用、是否需要练习或案例。

详细模板和检查清单见 [content-organization-canvas.md](references/content-organization-canvas.md)。

## 输出格式

除非用户要求其他格式，按以下结构输出：

1. 内容组织原则：本次内容取舍依据和主线逻辑。
2. 内容取舍表：来源内容项、对应目标/模块、处理方式、理由。
3. 内容分类表：通用原理、实践经验、操作方法、案例素材、练习任务、工作启发。
4. 模块内容组织表：模块、关键内容、表达逻辑、示例/案例、应用方式。
5. 缺口与补充建议：缺少的案例、数据、模板、专家经验或练习材料。
6. 后续过程设计提示：哪些内容适合讲授、讨论、案例、演练或工具练习。

快速请求时，可只输出“内容取舍表 + 模块内容组织表”。

## 质量规则

- 课程目标是内容裁剪的第一依据。
- 只保留服务目标学员工作应用的内容。
- 不按资料目录机械堆内容。
- 不把背景、制度原文或工具说明当成完整教学内容。
- 每个关键内容都要说明工作应用价值。
- 技能和判断类内容必须标注练习或案例承接。
- 通用原理要配实践经验，实践经验要提炼成可迁移方法。
- 如果内容过多，主动建议下沉为预习、附录、工具表或拆课。

## 与其他能力的关系

使用 `$topic-analysis` 先明确课题边界、学员场景和 KAS 内容。

使用 `$course-objectives` 先明确课程目标和目标水平。

使用 `$course-structure` 先确定模块结构和内容边界。

使用 `$content-organization` 对内容进行筛选、分类、知识结构梳理和表达逻辑建立。

内容稳定后，使用 `$process-design` 进行教学流程、教学方法、活动、案例、引导问题和学习衡量设计。
